# Contributing to the Docs

The easiest way to improve a page is to use the `Edit on GitHub` action on that page.

Use that path for:
- fixing commands
- clarifying setup steps
- adding short operational notes
- correcting wording or links

## Adding Images and Videos

Place media in:
- `data_pipeline/docs/assets/images/`
- `data_pipeline/docs/assets/videos/`

Prefer short, descriptive file names such as:
- `setup-operator-console-overview.png`
- `ops-zeus-usb-port-controller-map.png`
- `setup-operator-console-end-to-end-walkthrough.mp4`

Then link or embed the asset from the markdown page you are editing.

## Keep It Light

Prefer:
- compressed images
- short videos
- small, targeted edits

Avoid:
- very large raw recordings
- duplicate copies of the same media
- adding internal notes to the public docs
